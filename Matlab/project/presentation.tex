\section{Introduzione}

\begin{frame}
\frametitle{Dati della consegna}
\framesubtitle{Arrivi}
    Le richieste di arrivo sono casuali con un tasso medio $=\lambda$. Il tempo di arrivo tra un arrivo e il successivo segue una distribuzione esponenziale:
\[
t\_arrivo \sim Exp(\lambda)
\]
Di conseguenza
\[
\mathbb{E}[A]=\frac{1}{\lambda}
\]
\end{frame}

\begin{frame}
\frametitle{Dati della consegna}
\framesubtitle{Servizi}
   Ogni server impiega
\[
t\_servizio \sim Exp(\mu)
\]
Di conseguenza
\[
\mathbb{E}[S]=\frac{1}{\mu}
\]
\end{frame}

\begin{frame}
    \frametitle{Teoria delle code}

    La teoria delle code è lo studio matematico delle linee di attesa (o code) e di processi correlati, quali il processo di arrivo in coda, l'attesa  e il processo di servizio.
    Noi useremo la notazione di David George \underline{Kendall} in questa teoria.

\end{frame}

\begin{frame}
    \frametitle{Teoria delle code}
    \framesubtitle{Kendall}
    Kendall introdusse notazione $A/B/C$ per descrivere il modello a code.
    La prima lettera ($A$) indica la distribuzione del  sistema di arrivo, $B$ è la distribuzione dei servizi e la $C$ è il numero di server
    $A$ e $B$ sono distribuzioni di tempi, non di numeri.
    Nella prima lettera i valori comuni che possiamo avere sono: 
    \begin{itemize}
        \item $M\to$ arrivi Markoviani
        \item $D\to$ arrivi deterministici (intervalli fissi)
        \item $E_k\to$ arrivi per una distribuzione di Erlang ($[0,+\infty)$)
        \item $G\to$ generica distribuzione
    \end{itemize}
    Per $B$ valgono gli stessi valori.\\
    Seguendo le regole appena spiegate si deduce che abbiamo un sistema del tipo $M/M/1$
\end{frame}

\begin{frame}
    \frametitle{Teoria delle code}
    \framesubtitle{Markov}
    Markoviano vuol dire che soddisfa la proprietà di Andrey \underline{Markov}. Ovvero:
    \[
    P(futuro|presente,passato)=P(futuro|presente)
    \]
    La distribuzione esponenziale è l'unica distribuzione continua con la proprietà di assenza di memoria.
    La proprietà di assenza di memoria è quando la variabile T è senza memoria se:
    \[P(T>s+t|T>s)=P(T>t)
    \] 
    La formula indica che se ho già aspettato s, la probabilità di aspettare ancora t è la stessa di quando ho iniziato ad aspettare.
\end{frame}


\begin{frame}
    \frametitle{Teoria delle code}
    \framesubtitle{Markov = esponenziale}
    Quindi per l'esponenziale
    \[T\sim Exp(\lambda)\]
    \[P(T>t)=e^{-\lambda t}\]
    Sostituiamo:
    \[P(T>s+t|T>s)=\frac{P(T>s+t)}{P(T>s)}=\frac{e^{-\lambda(t+s)}}{e^{-\lambda s}}=e^{-\lambda t}=P(T>t)\]
    Quindi il markoviano è uguale all'esponenziale
\end{frame}

\begin{frame}
    \frametitle{Teoria delle code}
    \framesubtitle{Parametro di traffico}
    Nel modello M/M/N, secondo la Teoria delle code il parametro di traffico del sistema è
    \[\rho=\frac{\lambda}{N\cdot \mu}\]
    Dove:
    \begin{itemize}
        \item $N$ è il numero di server
        \item $\lambda$ è il tasso medio di arrivo (arrivi per unità di tempo)
        \item $\mu$ è il tasso medio di servizio (arrivi serviti per unità di tempo)
    \end{itemize}

    % Inoltre nel modello M/M/1:
    % \begin{itemize}
    %     \item Numero medio di arrivi nel sistema:
    %     \[L=\frac{\rho}{1 - \rho}\]
    %     \item Tempo medio nel sistema:
    %     \[W=\frac{1}{\mu - \lambda}\]
    % \end{itemize}
    Per trovare il limite di N basta eseguire la seguente equazione:
    \begin{align}
        rho&=\frac{\lambda}{N\cdot \mu}\\
        1&=\left \lceil \frac{\lambda}{N\cdot \mu} \right \rceil\\
        N&=\left \lceil \frac{\lambda}{\mu} \right \rceil
    \end{align}

\end{frame}

\begin{frame}
        \frametitle{Teoria delle code}
        \framesubtitle{Parametro di traffico}
        Dove $\left \lceil\right \rceil$ è la funzione \texttt{ceil()}. Inoltre
        \begin{itemize}
            \item Se $\rho < 1$: il sistema è stabile, la coda non cresce all'infinito
            \item Se $\rho \geq 1$: la coda cresce indefinitamente, il sistema è sovraccarico 
        \end{itemize}
        
\end{frame}

\begin{frame}
    \frametitle{distribuzione esponenziale}
    Noi possiamo usare distribuzioni uniformi \texttt{rand()} o normali standard \texttt{randn()}. Quindi per l'esponenziale dobbiamo trovare un modo per calcolarlo.
    Utiliziamo il metodo dell'inversa.
    Se abbiamo un $U \sim Unif(0,1)$ cioè un numero tra 0 e 1.L'idea è che conosco la CDF ($F(x)$) posso invertirla per trasformare $U$ nella mia esponenziale
\end{frame}

\begin{frame}
    \frametitle{distribuzione esponenziale}
    La CDF dell'esponenziale è
    \[F_X(x)=\begin{cases}
        0,& \mbox{Se} x>0\\
        1-e^{-\lambda x},& \mbox{Se} x\geq 0
    \end{cases}\]
    La PDF è:
    \[Exp(\lambda,x)=\begin{cases}
        0,& \mbox{Se} x>0\\
        \lambda e^{-\lambda x},& \mbox{Se} x\geq 0
    \end{cases}\]
    Verifichiamo anche facendo l'integrale della PDF per trovarmi la CDF
    \[F_X(x)=\int_0^x{\lambda e^{-\lambda x}}= 1-e^{-\lambda x}\]
\end{frame}

\begin{frame}
    \frametitle{distribuzione esponenziale}
    Quindi possiamo porre la seguente equazione (dove U è l'uniforme):
    \begin{align}
        U&=F(x)\\
        U&=1-e^{-\lambda x}\\
        e^{-\lambda x}&=1-U\\
        -\lambda x&=\ln{(1-U)}\\
         x&=-\frac{\ln{(1-U)}}{\lambda}
    \end{align}
    Ecco la formula per passare dall'uniforme all'esponenziale. Ma possiamo scriverla anche come
    \[-\frac{\ln{(U)}}{\lambda}\]
    perchè sia $1-U$ che $U$ sono entrambi uniformi tra 0 e 1
\end{frame}

\begin{frame}

    \framesubtitle{analisi statistica}
    Ripetendo la simulazione K volte si ottengono K stime del tempo medio di attesa. Da lì si può:
    \begin{itemize}
        \item Calcolare media e varianza campionarie
        \item Costruire intervalli di confidenza al $95\%$ usando la distribuzione t di Student (per K piccolo) o normale (per K grande)
        \item Verificare il Teorema del Limite Centrale: le medie campionarie si distribuiscono normalmente, anche se il tempo di attesa singolo non lo è
    \end{itemize}
\end{frame}

\begin{frame}{Metodo Monte Carlo}
Consiste in  simulare il sistema evento per evento, invece di risolvere equazioni analitiche. La logica è:
\begin{itemize}
    \item Genera il tempo del prossimo arrivo
    \item Genera il tempo di servizio per quella richiesta
    \item Assegna la richiesta al primo server libero, oppure mettila in coda
    \item Avanza il tempo al prossimo evento (arrivo o fine servizio)
    \item Ripeti fino al tempo T
\end{itemize}

Questo approccio si chiama simulazione a eventi discreti

\end{frame}

\begin{frame}
    \frametitle{Funzione queue lenght}
    Nel codice si ha:\\
    \texttt{future\_load = sum(max(0, server\_free\_at - current\_time));}\\
    \texttt{approx\_queue = max(0, round(future\_load * mu) - N);}\\
    Perchè si moltiplica per $\mu$?
    perchè $mu$ nel sistema $M/M/N$ è \[\mu=\text{tasso di servizio}=\frac{\text{Arrivi}}{\text{Unità di tempo}}\]
    quindi:
    \begin{align}
        future_load&\cdot \mu=\\
        (\text{Unita di tempo})&\cdot \frac{\text{Arrivi}}{\text{Unità di tempo}}\approx \text{Arrivi}
    \end{align}
\end{frame}

\begin{frame}
    \frametitle{Funzione polyfit}
    \texttt{polyfit(x,y,n)} trova il polinomio di grado $n$ che approssima meglio i dati nel senso dei minimi quadrati. Dove $x$ e $y$ sono le ascisse e le ordinate dei dati.
    $n$ in questo caso è 1 quindi la nostra è una regressione lineare e non polinomiale.
    Per comprendere meglio supponiamo di avere i dati
    \[
    (x_1, y_1), (x_2, y_2), \dots, (x_m, y_m)
    \]
    e si vuole trovare un polinomio di grado $n$:
    \[
    p(x) = a_n x^n + a_{n-1} x^{n-1} + \dots + a_1 x + a_0
    \]
    La funzione \texttt{polyfit} cerca i coefficienti $a_n, \dots, a_0$ che \textbf{minimizzano l'errore quadratico totale}:
    \[
    S = \sum_{i=1}^{m} (y_i - p(x_i))^2 
    = \sum_{i=1}^{m} \big( y_i - (a_n x_i^n + \dots + a_0) \big)^2
    \]
\end{frame}

\begin{frame}
    \frametitle{Funzione polyfit}
    \framesubtitle{funzione obbiettivo}

    Dove:
    \begin{itemize}
        \item $y_i$ è il valore reale del dato
        \item $\hat{y}_i = p(x_i)$ è il valore predetto dal polinomio
    \end{itemize}

    Per ciascun dato, si definisce l'errore:

    \[
    e_i = y_i - \hat{y}_i
    \]

    La funzione obiettivo (somma dei quadrati degli errori) è:

    \[
    S = e_1^2 + e_2^2 + \dots + e_m^2
    \]

    Più $S$ è piccolo, meglio il polinomio approssima i dati.  
    L'uso del quadrato serve a penalizzare maggiormente gli scostamenti grandi rispetto a quelli piccoli.
\end{frame}
